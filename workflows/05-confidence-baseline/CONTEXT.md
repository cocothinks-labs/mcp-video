# 05-confidence-baseline

Generate a tiny synthetic source clip, run a staged social-video workflow, produce a quality report, and write a Video Receipt.

Use this workflow when you need proof that mcp-video can move from install to reviewed output without relying on private media.

## Inputs

| Source | File/Location | Description |
| --- | --- | --- |
| Synthetic source | Generated by `workflow.py` if no input is provided | 6-second FFmpeg test pattern with sine audio |
| Optional source video | First CLI argument | Any local video file |
| Receipt schema | `docs/VIDEO_RECEIPT.md` | Trust artifact fields |

## Process

1. **00-source**: Generate a small source clip if no input is provided.
2. **01-trim**: Trim to the proof segment.
3. **02-resize**: Resize to `9:16`.
4. **03-caption**: Add proof hook text.
5. **04-normalize**: Normalize audio to `-14 LUFS`.
6. **05-export**: Export final MP4.
7. **06-checkpoint**: Run quality check and release checkpoint.
8. **07-receipt**: Write `video_receipt.json`.

## Outputs

| Artifact | Location | Format |
| --- | --- | --- |
| Source clip | `output/source.mp4` | MP4 |
| Trimmed clip | `output/01_trimmed.mp4` | MP4 |
| Vertical clip | `output/02_vertical.mp4` | MP4 |
| Captioned clip | `output/03_captioned.mp4` | MP4 |
| Normalized clip | `output/04_normalized.mp4` | MP4 |
| Final clip | `output/final_clip.mp4` | MP4 |
| Quality report | `output/quality.json` | JSON |
| Release checkpoint | `output/checkpoint/` | JSON + thumbnail/storyboard |
| Video Receipt | `output/video_receipt.json` | JSON |

## Quality Gates

- [ ] Required dependencies pass `mcp-video doctor`.
- [ ] Final video is `9:16`.
- [ ] Audio passes quality check.
- [ ] Release checkpoint creates thumbnail and storyboard artifacts.
- [ ] Video Receipt records tool calls, quality, warnings, and human-review status.
